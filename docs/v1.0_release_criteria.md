# AllBrain MCP v1.0 Release Criteria

This document defines the measurable criteria that must be satisfied before
AllBrain MCP can be released as v1.0. Each criterion has a clear threshold
and verification method.

## 🧪 Test Coverage & Quality

- **Code coverage:** ≥ 85% line coverage (measured via `pytest-cov`)
- **Branch coverage:** ≥ 80% branch coverage
- **Test pass rate:** 100% of non-skipped tests pass
- **Static analysis:** Ruff lint and format checks pass with zero warnings
- **Type checking:** Pyright basic mode passes with zero errors

## 🔧 Functional Requirements

- **MCP tool completeness:** All 51 documented MCP tools are functional and tested
- **Decision pipeline:** 4-step pipeline (Preparation → Reasoning → Feedback → Learning) operates end-to-end
- **Conflict resolution:** Multi-agent conflict detection and resolution works at basic level (ConflictDetector + ConflictResolver)
- **Event sourcing:** Event append, replay, and snapshot generation work correctly
- **Session management:** Session start, claim, renew, and close lifecycle is stable

## 🔄 Backward Compatibility

- **Legacy shim removal:** All v0.4.x shim paths (`allbrain.<module>`) still import correctly or emit clear deprecation warnings
- **API stability:** No breaking changes to public tool signatures without deprecation notice
- **Migration path:** Existing v0.4.x databases can be upgraded without data loss
- **Import compatibility:** `from allbrain.<module> import <Class>` works for all migrated modules

## 📊 Performance

- **Startup time:** Server starts within 5 seconds on reference hardware
- **Event throughput:** ≥ 1000 events/second append rate
- **Snapshot generation:** ≤ 10 seconds for 10,000 events
- **Memory usage:** ≤ 512 MB RSS under normal load

## 🔒 Security

- **Secret redaction:** All API keys and tokens are redacted in logs and outputs
- **Input validation:** All tool inputs are validated against schemas
- **Path traversal:** No path traversal vulnerabilities in file operations
- **Rate limiting:** Rate limiter prevents abuse (process-local acceptable for v1.0)

## 📚 Documentation

- **Architecture docs:** ARCHITECTURE.md is up-to-date with current module structure
- **Tool documentation:** All 51 MCP tools have clear descriptions and examples
- **Setup guide:** setup.md covers installation, configuration, and first run
- **Migration guide:** upgrade.md documents breaking changes and migration steps

## 🏗️ Migration Status

- **Reasoning context:** ✅ Migrated (10 modules)
- **Analysis context:** ✅ Migrated (17 modules)
- **Governance context:** ⏳ Pending (12 modules) — target v0.4.2
- **Learning context:** ⏳ Pending (12 modules) — target v0.4.2
- **Collaboration context:** ⏳ Pending (10 modules) — target v0.4.3
- **Memory context:** ⏳ Pending (12 modules) — target v0.4.3

**v1.0 requires:** All 6 bounded contexts migrated to `allbrain.domains.*` namespace.

## 🐛 Bug Criteria

- **Critical bugs:** 0 known critical bugs (data loss, security vulnerabilities, crashes)
- **High-severity bugs:** 0 known high-severity bugs affecting core functionality
- **Known issues:** Only low-severity or environmental issues documented in CHANGELOG

## ✅ Release Readiness Checklist

- [ ] All test coverage thresholds met
- [ ] All functional requirements verified
- [ ] Backward compatibility tests pass
- [ ] Performance benchmarks meet thresholds
- [ ] Security audit completed
- [ ] Documentation reviewed and updated
- [ ] All bounded contexts migrated
- [ ] No critical or high-severity bugs open
- [ ] CHANGELOG updated with v1.0 changes
- [ ] Version bumped to 1.0.0 in pyproject.toml
